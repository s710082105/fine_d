# Codex 主工作台设计

## 目标

把当前 `Codex` 页从“终端承载页”提升为 FineReport 主工作台入口，但保持执行主体仍然是右侧真实 `codex` 终端，而不是页面内流程编排器。

本轮目标有三项：

- 打开 `Codex` 页时，为当前项目自动准备一套可执行的上下文与 FineReport 专用 skill。
- 在同一页左侧展示远程数据连接和远程文件清单，并支持点击插入终端，辅助用户和 Codex 操作。
- 让 Codex 在终端内自行完成“需求确认 -> 检查远端 -> 拉取/补全 -> 本地修改 -> 同步推送 -> 浏览器复核 -> 最终报告”的完整闭环。

## 方案选择

本轮采用 `B 方案`：

- 页面不实现流程编排状态机。
- 页面不自动代替 Codex 执行同步、拉取、预览。
- 页面只负责准备上下文、承载真实终端、提供左侧辅助信息和最小插入能力。

不采用以下方案：

- 不在页面里增加“下一步”“执行流程”等向导式控制。
- 不在前端或后端单独维护一套工作流执行器与报告生成器。
- 不把终端退化成日志输出区域。

## 范围边界

本轮覆盖：

- `Codex` 页进入前的项目上下文生成
- 项目级 `AGENTS.md` 生成与手动刷新
- 左侧远程数据连接与远程文件清单
- 点击插入终端
- 右侧真实 Codex 终端承载
- 页面级错误暴露和状态提示

本轮不覆盖：

- 页面内自动步骤编排
- 页面内结构化任务卡
- 页面替 Codex 自动发起同步、拉取或预览
- 多终端、多项目并行会话

## 页面结构

`Codex` 页调整为固定双栏：

### 左侧协作栏

左侧由三块组成：

1. `项目上下文状态`
   - 当前项目目录
   - 当前远程参数摘要
   - 最近一次上下文生成时间
   - `重新生成上下文` 按钮

2. `远程数据连接`
   - 展示连接名和数据库类型
   - 点击后将连接名或预设片段插入终端输入流

3. `远程文件清单`
   - 展示远程 `reportlets` 范围内的文件和目录
   - 支持懒加载
   - 点击后将远程路径、目标文件名或操作提示片段插入终端输入流

左侧的职责是辅助，不承担执行控制。所有按钮都只能做“刷新”“插入”“重新生成上下文”这类显式动作。

### 右侧终端区

右侧继续承载真实 `codex` 终端：

- 使用终端仿真器渲染 PTY 输出
- 原样传递键盘流
- 自动创建会话
- 显式展示会话状态与错误

页面层不解析 Codex 输出语义，不把输出拆分成聊天气泡，也不在页面里额外判断当前流程走到哪一步。

## 上下文生成

打开 `Codex` 页时，系统在真正创建终端前先生成项目级上下文文件。

生成内容分为四层：

### 1. 项目级 `AGENTS.md`

项目根生成或覆盖 `AGENTS.md`，作为最终导引文件。

职责：

- 明确当前项目目录与执行范围
- 固定 FineReport 工作顺序
- 声明必须使用的上下文文件和 skill
- 约束最终必须产出准确报告
- 明确不允许跳过远端检查、同步推送和浏览器复核

`AGENTS.md` 的定位高于普通上下文说明，用来防止 Codex 执行跑偏。

### 2. `.codex/project-context.md`

记录当前项目事实：

- 当前项目目录
- 远程服务地址与设计器运行目录
- 当前远程目录摘要
- 当前数据连接摘要
- 本地 `reportlets`、`templates` 等关键目录说明

### 3. `.codex/project-rules.md`

记录执行规则：

- 先确认用户需求
- 涉及报表先检查远端
- 必要时先拉取或补全远端文件
- 修改后必须同步推送
- 同步后必须浏览器复核
- 最后必须输出准确报告

### 4. `.codex/skills/*`

写入或刷新 FineReport 专用 skill 入口，至少覆盖：

- 报表创建
- 数据库扫描
- 远端文件检查
- 远端文件拉取
- 远端文件补全
- 同步推送
- 浏览器复核

## `AGENTS.md` 更新策略

`AGENTS.md` 不在每次打开 `Codex` 页时自动重写。

采用用户确认后的策略：

- 页面首次进入时，如果目标文件不存在，则自动生成
- 页面提供 `重新生成上下文` 按钮
- 只有用户主动触发时，才覆盖并刷新 `AGENTS.md`、`.codex/project-context.md`、`.codex/project-rules.md` 和相关 skill 文件

这样可以避免用户在项目中已有自定义规则时被静默覆盖。

## 执行流程

本轮的完整执行链路由 Codex 在终端内自主完成：

1. 页面生成上下文
2. 页面创建真实 Codex 会话
3. 用户在终端内描述需求
4. Codex 结合 `AGENTS.md`、上下文文件和 skill 先确认需求
5. 需求确认完成后，Codex 自主执行：
   - 检查远端文件
   - 按需补全或拉取
   - 本地修改
   - 同步推送
   - 浏览器复核
6. Codex 在终端内输出最终报告

页面不做以下判断：

- 不判断需求是否已经确认
- 不判断现在是否该拉取远端文件
- 不判断是否应该开始同步
- 不生成独立于 Codex 之外的第二份结构化报告

最终报告以 Codex 终端输出为准。

## 左侧插入策略

左侧点击插入是为了降低用户操作成本，不是替代终端。

### 数据连接插入

点击数据连接项时，默认插入连接名；若后续上下文里已定义标准数据库扫描提示模板，也可以插入一个短提示片段，例如“请先扫描连接 `<name>` 的表结构”。

### 远程文件插入

点击文件项时，默认插入远程路径；点击目录项时，默认插入目录路径。

插入行为必须满足：

- 不直接自动执行
- 不自动补回车
- 不改变当前终端状态
- 仅写入当前输入流，保留用户继续编辑的空间

## API 与模块调整

本轮需要新增或扩展两类能力：

### 后端

- 新增“上下文生成”用例与接口
- 在 `Codex` 会话创建前可显式调用上下文生成
- 新增远程文件清单接口，供 `Codex` 页左侧使用
- 保持现有终端会话 API，不新增页面流程编排接口

### 前端

- `Codex` 页拆成左侧协作栏 + 右侧终端区
- 新增“重新生成上下文”操作
- 新增“点击插入终端”桥接
- 复用现有远程目录和连接展示能力，但重组为 Codex 工作台左栏形态

## 失败暴露

保持现有“显式失败”原则：

- 当前项目目录不存在：阻止生成上下文和创建终端
- 远程参数不完整：阻止上下文生成，并明确提示缺失项
- `AGENTS.md` 或 `.codex/*` 写入失败：显式报错，不继续创建终端
- 远程文件清单加载失败：左侧报错，但不影响已经创建好的终端继续使用
- 终端创建失败：直接暴露错误，不做假终端
- 插入终端失败：明确提示，不静默吞掉

## 测试要求

本轮设计要求至少覆盖以下验证：

- 上下文生成接口在项目目录有效时能正确写出文件
- `AGENTS.md` 仅在首次缺失或点击“重新生成上下文”时更新
- `Codex` 页挂载时会先请求上下文生成，再创建终端
- 左侧远程数据连接和远程文件清单能正常展示
- 点击连接或文件时，内容被写入当前终端输入流
- 终端仍保持真实 ANSI 渲染，不回退到纯文本日志面板
- 构建、类型检查和前端测试保持通过

## 最终结果

交付完成后，`Codex` 页应成为 FineReport 主工作台入口：

- 用户进入页面即可获得最新项目上下文
- Codex 被项目级 `AGENTS.md` 和 skill 约束在正确轨道内
- 用户既可以完全依赖 Codex 自主完成全流程，也可以通过左侧快速插入连接名、远程路径来辅助操作
- 最终执行报告由 Codex 在终端内输出，页面不再额外制造第二套流程语义
