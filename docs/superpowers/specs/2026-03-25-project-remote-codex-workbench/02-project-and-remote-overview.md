# 02. 项目与远程概览页

## 页面职责

页面 1 固定承担三件事：

- 选择当前项目目录
- 维护该项目的远程参数
- 展示基于该远程参数拉回来的远程概览

它不承担 Codex 会话交互，不承担同步动作，不承担报表编辑。

## 页面结构

页面分成三块：

### A. 项目目录区

元素：

- 当前项目目录展示
- `选择目录` 按钮
- `重新载入` 按钮

规则：

- 目录由系统目录选择器返回，不允许手输路径作为首版主交互。
- 选择目录后，页面立即切换当前项目上下文。
- 若该项目已有保存过的远程参数，页面同步回填。

### B. 远程参数区

字段固定为：

- `服务地址`
- `用户名`
- `密码`

按钮：

- `保存参数`
- `测试连接`

规则：

- 首版不加端口、远程根目录、数据连接名等额外字段。
- `保存参数` 只保存到当前项目目录对应的配置。
- `测试连接` 不写入其它业务状态，只返回连接结果。

### C. 远程概览区

远程概览分成两个并列面板：

- `远程目录`
- `数据连接`

规则：

- 两个面板使用同一组远程参数。
- 首版由一个统一刷新动作驱动，不要求前端分别维护两套请求生命周期。
- 面板只读，不在这一页直接编辑远端文件或连接。

## 状态流

### 初始进入

1. 读取当前项目状态。
2. 若还没选项目目录，显示空状态提示。
3. 若已选项目目录，则加载该项目已保存的远程参数。

### 选择项目目录

1. 用户点击 `选择目录`。
2. 系统弹出目录选择器。
3. 后端持久化“当前项目目录”。
4. 前端刷新项目区和远程参数区。
5. 若远程参数完整，可允许刷新远程概览。

### 保存远程参数

1. 用户填写 `服务地址 / 用户名 / 密码`。
2. 点击 `保存参数`。
3. 后端将参数绑定到当前项目目录。
4. 页面更新本地状态，不额外刷新其它面板。

### 加载远程概览

1. 用户点击 `测试连接` 或 `刷新概览`。
2. 后端使用当前项目绑定的远程参数获取目录与数据连接。
3. 两个面板同时刷新。

## 错误处理

- 未选择项目目录：显式提示“请先选择项目目录”。
- 远程参数不完整：禁止测试连接和远程概览加载。
- 目录读取失败：项目区提示真实错误。
- 远程连接失败：远程参数区或远程概览区直接显示后端错误。
- 远程目录失败和数据连接失败：允许按面板分别暴露错误，不互相吞掉。
