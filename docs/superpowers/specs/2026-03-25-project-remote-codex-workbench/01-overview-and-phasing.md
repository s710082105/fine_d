# 01. 总览与阶段划分

## 背景

当前前端主壳已经有 `Project / Datasource / Reportlet / Sync / Preview / Assistant` 六个视图，但用户真正需要的不是只读演示，而是一条可操作的工作台主链路：

1. 明确当前正在操作哪个本地项目目录。
2. 给该项目配置可工作的远程 FineReport 连接参数。
3. 用同一组远程参数查看远程目录与远程数据连接。
4. 在页面内直接打开 Codex 终端继续工作。

这条链路要求页面职责清楚，且不能把配置、浏览、终端三类状态揉成一个超大页面。

## 设计结论

工作台首版按两个页面落地：

- 页面 1：`项目与远程概览`
- 页面 2：`Codex`

页面 1 负责“选择哪个项目”和“看这个项目现在连到了哪里”。页面 2 只负责“在这个项目上下文里打开一个新的 Codex 终端会话”。

## 分阶段执行

### 阶段 1：项目与远程概览

目标：

- 用目录选择框选择当前项目目录。
- 为该项目保存 `服务地址 / 用户名 / 密码`。
- 在同一页展示远程目录与数据连接两个面板。

### 阶段 2：Codex 终端页

目标：

- 独立页面打开 Codex 终端。
- 每次进入页面都创建一个新会话。
- 页面不自研聊天交互，只承载终端与最小会话状态。

## 非目标

- 不做项目列表。
- 不做远程 profile 历史记录。
- 不做“项目页里直接夹一个聊天区”。
- 不让前端自行拼装 Codex 消息流。
- 不对远端失败做 silent fallback。

## 验收口径

阶段 1 和阶段 2 完成后，用户应能走通以下最小路径：

1. 进入页面。
2. 选择一个本机项目目录。
3. 保存该项目的远程连接参数。
4. 在页面内看到远程目录和远程数据连接。
5. 切到 `Codex` 页面。
6. 页面内出现一个新的 Codex 终端会话。
