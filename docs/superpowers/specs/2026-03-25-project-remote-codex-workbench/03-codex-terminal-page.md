# 03. Codex 终端页

## 设计原则

Codex 页首版不是“聊天页面”，而是“终端承载页面”。

系统不自研以下能力：

- 消息气泡
- 输入框 + 发送按钮
- 消息协议转换
- 对终端输出做语义拆分

系统只负责在页面里承载一个真正的 Codex 终端会话。

## 页面职责

Codex 页只承担四件事：

- 创建会话
- 展示终端
- 暴露最小会话状态
- 允许用户重开会话

## 会话模型

首版固定规则：

- 每次进入 `Codex` 页面，都新建一个终端会话。
- 不复用旧会话。
- 不支持会话列表。
- 离开页面后，不要求保活复用。

## 页面结构

页面元素保持最小：

- 当前项目目录
- 当前会话状态
- `重新创建会话` 按钮
- 大块终端区域

终端区域占页面主体，视觉上应是核心元素，不被表单和侧栏挤压。

## 上下文边界

Codex 会话只自动继承：

- 当前项目目录作为工作目录

首版不自动注入：

- 远程参数
- 远程目录结果
- 数据连接结果

这些能力以后若要补，也应通过明确的上下文注入策略实现，而不是先写死到终端启动命令里。

## 错误处理

需要显式暴露以下错误：

- Codex 未安装
- PTY 创建失败
- 工作目录不存在
- 会话启动命令失败
- 终端流读取失败

页面不做假终端，也不返回“模拟启动成功”。

## 技术实现方向

后端需要暴露终端会话生命周期接口：

- 创建会话
- 读取输出
- 写入输入
- 关闭会话

前端只负责：

- 挂载终端组件
- 与后端会话接口通信
- 把输入和输出直接桥接到终端

这意味着页面和终端协议之间仍有前后端桥接，但页面层不再自行发明一套聊天交互。
