# 终端会话生命周期

## 创建

1. 用户先选择并保存项目配置。
2. 右侧显示“启动 Codex”按钮。
3. 点击后，前端调用 `create_terminal_session`。
4. Rust 校验项目目录存在且为绝对路径。
5. Rust 创建 PTY，并在项目目录中执行 `codex`。
6. 前端收到创建成功事件后挂载终端并进入运行态。

## 输入与输出

- 键盘输入通过 `write_terminal_input` 写入 PTY。
- PTY 原始输出通过事件推送回前端，由 `xterm.js` 原样渲染。
- 窗口尺寸变化时，前端调用 `resize_terminal`。

## 关闭与退出

- 用户点击“关闭终端”时，前端调用 `close_terminal_session`。
- Codex 自行退出时，后端推送 `exited` 事件和退出码。
- 终端退出后，右侧保留最后一屏内容，但状态变为“已退出”。

## 项目切换

- 当前终端运行中时切项目，旧终端必须先关闭。
- 关闭完成后，右侧回到“启动 Codex”状态。
- 不允许把 A 项目的终端 session 带到 B 项目。

## 配置变更

- 左侧保存成功后，如果当前终端运行中，右侧显示“配置已更新，请重启 Codex 终端”。
- 不做静默自动重启，避免打断用户正在输入的命令。

## 失败暴露

- 项目目录未选择：阻止启动并显示明确错误。
- 本机未安装 `codex`：显示安装提示。
- PTY 创建失败、进程启动失败、写入失败、resize 失败：全部原样显示到右侧状态区。
- 不做 mock 终端，不返回假成功。
