# UI 与状态模型

## 右侧 UI 结构

右侧改为三段结构：

1. 状态条
说明：显示项目名、终端状态、PID、配置变化提示。

2. 终端区域
说明：由 `xterm.js` 挂载，填满右侧主体。

3. 操作条
说明：只保留三个操作：
- `启动 Codex`
- `重启终端`
- `关闭终端`

## 状态模型

前端状态收敛为：

- `idle`
- `starting`
- `running`
- `exited`
- `error`

同时记录：

- `terminalSessionId`
- `projectId`
- `pid`
- `exitCode`
- `isConfigStale`

## 交互规则

- `idle`：只允许启动。
- `starting`：禁用全部重复启动操作。
- `running`：允许重启、关闭；终端接收键盘输入。
- `exited`：允许重启；保留终端输出。
- `error`：允许重新启动；错误信息保留在状态区。

## 替换范围

下列右侧内容全部移除：

- 会话信息卡片
- Codex 输出文本区
- 消息输入框
- 新建会话按钮
- 刷新上下文按钮
- 中断会话按钮

它们由终端状态条和终端控制按钮替代。
