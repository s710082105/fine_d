# Status Check Examples

本文件给 `fr-status-check` 提供示例索引，不在 `SKILL.md` 展开长说明。

## Example Map

- `references/checklist.md`
  - 何时看：需要逐项核对 doctor 证据
  - 重点：配置、Designer Java、bridge、Decision、connection、remote reportlets
- `../fr-db/EXAMPLES.md`
  - 何时看：状态检查通过后，下一步是数据源、SQL、字段探测
  - 重点：Decision HTTP 范围和面向 CPT/FVS 的数据证据
- `../fr-download-sync/EXAMPLES.md`
  - 何时看：状态检查通过后，下一步要先拉远端模板
  - 重点：同步前提和后续样本入口
- `../fr-create/EXAMPLES.md`
  - 何时看：状态检查通过后，需要新建一个 reportlet
  - 重点：冲突检查、模板选择和后续编辑入口

## Usage

- 先按 `checklist.md` 做最小可运行性取证
- 再按“探数据 / 拉远端 / 新建模板”转到后续 skill 的示例入口
- 状态检查本身不负责展开模板结构细节
