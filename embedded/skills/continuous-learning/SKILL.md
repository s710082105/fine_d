# 持续学习

当客户对当前结果提出问题，并且给出了学习样本、对照案例、正确结果或约束说明时，必须把这次反馈转成可复用的 skill 更新。

## 执行规则

1. 先确认这次反馈暴露的差异点，区分是需求理解偏差、模板结构遗漏、数据口径错误，还是验证流程缺失。
2. 对照已有 `skills/`，定位最相关的 skill；如果没有合适归属，就新增一个 skill，而不是把经验散落在临时回复里。
3. 把学习样本总结成稳定规则，写进 skill 的 `SKILL.md`：
   - 触发条件
   - 正确做法
   - 必要的反例或禁忌
   - 样本中能复用的关键结构
4. 更新 skill 后，再继续当前任务，确保新的规则已经用于这次修正。

## 写入要求

- 只沉淀能跨任务复用的经验，不要把一次性的临时路径或脏数据原样塞进 skill。
- 如果样本带来新的验证要求，要把验证动作一并写进 skill。
- 不允许只口头说明“下次注意”，必须真的改 `skills/` 文件。
