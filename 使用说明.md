# FineReport Runtime 使用说明

本文档单独提供给部署包用户阅读，安装、配置、授权和自动改进都以本文件为准。

## 1. 前置条件

软件使用前，宿主机至少需要满足以下条件：

| 条目 | 要求 | 检查方式 |
| --- | --- | --- |
| Python | `Python 3.11` 或更高版本 | `python3 --version` |
| Node.js / npm | 宿主机可用 `node` 和 `npm` | `node --version`、`npm --version` |
| Codex CLI | 宿主机可直接执行 `codex` 命令 | `codex --version` |
| FineReport Designer | 已安装本机设计器，且后续 `designer_root` 指向真实安装目录 | 确认安装目录存在，例如 `/Applications/FineReport` |
| Decision 服务 | 能访问目标 `decision_url`，并已知登录账号密码 | 浏览器打开 `http://<host>/webroot/decision` |
| 浏览器 | 如需浏览器复核，宿主机需能打开 Chromium / Chrome 系浏览器 | 本机可正常打开浏览器 |

补充说明：

- 本仓 Python 运行时要求来自 `pyproject.toml` 的 `requires-python = ">=3.11"`。
- `node` / `npm` 主要用于 Codex CLI 与浏览器相关能力。
- `designer_root` 必须是设计器安装根目录，不要填工作区目录。

## 2. 配置说明

运行前先参考 `.codex/fr-config.demo.json` 创建真实配置文件 `.codex/fr-config.json`。

推荐步骤：

1. 复制 `.codex/fr-config.demo.json` 为 `.codex/fr-config.json`
2. 将占位值替换为真实环境
3. 运行状态检查：

```bash
python3 .codex/skills/fr-status-check/scripts/run.py --config-path .codex/fr-config.json
```

字段说明如下：

| 字段 | 是否必填 | 说明 | 典型示例 |
| --- | --- | --- | --- |
| `project_name` | 否 | 项目标识；为空时回退为 `workspace_root` 目录名 | `demo-workspace` |
| `decision_url` | 是 | Decision 根地址，只填登录入口，不填具体报表预览 URL | `http://127.0.0.1:8075/webroot/decision` |
| `designer_root` | 是 | FineReport Designer 安装根目录 | `/Applications/FineReport` |
| `username` | 是 | Decision 登录账号 | `admin` |
| `password` | 是 | Decision 登录密码 | `change-me` |
| `workspace_root` | 是 | 当前工作区根目录，必须是真实存在的本机路径 | `/Users/yourname/data/finereport-workspace` |
| `remote_root` | 否 | 远端根目录，只能位于 `reportlets` 下；留空默认 `reportlets` | `reportlets/demo` |
| `task_type` | 否 | 当前任务说明，仅用于记录 | `报表联调` |

填写规则：

- `.demo.json` 中所有 `_comment` 字段都只是注释，运行时不会读取。
- `decision_url` 必须是 Decision 登录入口，例如 `/webroot/decision`，不要写成 `view/report?...`。
- `designer_root` 必须指向本机设计器安装目录，运行时会在这里查找 Java 和 FineReport jars。
- `workspace_root` 必须指向当前实际工作区，不要填别人的路径，也不要填不存在的目录。
- 真实密码写入 `.codex/fr-config.json` 即可，不要把真实密码提交到仓库。

## 3. 授权说明

如果 bridge 未授权，程序会直接返回类似下面的消息：

```text
设备 MAC: AA:BB:CC:DD:EE:FF，请联系管理员授权
```

处理方式：

1. 复制报错中的完整 `MAC` 地址
2. 将该 `MAC` 发给管理员申请授权
3. 管理员生成 `fr-remote-bridge.auth` 后，放到 `fr-remote-bridge.jar` 同级目录
4. 重新执行原来的操作

注意：

- 没有授权时不要继续排查同步、上传、预览或其他 bridge 相关步骤。
- 如果出现 `请联系管理员授权`，按 bridge 原样消息处理即可。
- 如果授权文件与当前设备 MAC 不匹配，仍会继续报未授权。

## 4. 自动改进教程

如果程序把某个需求写错了，而你已经在设计器里把正确内容改好并保存到了正确的报表文件中，可以直接输入下面这句提示词，让软件回读正确报表并更新对应 skill：

```text
xxx软件的xxx需求编写错误，我已经将正确的编写内容放到了xxxx.cpt 阅读并学习后更新对应的skill下的example及 skill.md 为后续开发提供指导
```

使用说明：

- 第一个 `xxx软件` 替换为当前软件或项目名称。
- 第二个 `xxx需求` 替换为这次写错的需求名称。
- `xxxx.cpt` 替换为已经被你修正好的真实报表路径，例如 `reportlets/销售/订单汇总.cpt`。
- 如果修正的是 `.fvs`，把文件名替换成真实 `.fvs` 路径即可。

推荐做法：

1. 先在设计器里把正确内容改好并保存。
2. 确认修正后的报表文件已经落在当前工作区。
3. 再把上面的提示词发给软件，让它读取正确报表并更新 skill 示例。

这样后续同类需求会优先参考你刚刚修正过的真实模板，而不是继续沿用旧的错误写法。
